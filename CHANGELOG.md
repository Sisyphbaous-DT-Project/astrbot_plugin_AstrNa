# 更新日志

## 1.2.9

- 修复当前 Reply 引用图片补齐仍可能不稳定的问题：当引用组件里只有 NapCat / aiocqhttp 的本地临时图片路径，且该路径已经失效时，AstrNa 会丢弃这条死路径，并通过 `get_msg` / `get_image` 等 OneBot 接口重新解析可用图片。
- 增强 OneBot 兼容兜底：支持 `message_id` / `id` 两类取消息参数，并尝试 `get_file`、`get_group_file_url`、`get_private_file_url` 解析文件型图片引用。
- 该修复只处理当前消息 Reply 相关图片，不影响普通直接发图、历史图片清理、小模型群聊上下文压缩、图像转述和会话历史。

## 1.2.8

- 修复 NapCat / aiocqhttp 下引用图片视觉输入优化可能拿不到图片的问题：当 AstrBot 引用图解析器只识别 `bot.api.call_action`，而当前事件只有 `bot.call_action` 时，AstrNa 会提供轻量兼容代理。
- 该修复不修改 AstrBot Core、不 monkeypatch 原 event，只在 AstrNa 解析当前 Reply 引用图片时生效；当前直接发图、历史图片清理、小模型压缩和图像转述逻辑保持不变。
- 增加引用图补齐诊断日志：成功补图会记录追加数量，发现 Reply 但没有图片时记录 debug，便于排查过期引用或平台未返回图片的情况。

## 1.2.7

- 新增优化引用图片视觉输入：主动回复、影芯或其他第三方插件通过 `event.request_llm()` 自建请求时，如果当前 `Reply` 引用图片没有进入 `req.image_urls`，AstrNa 会在请求层调用 AstrBot 自带引用图解析并去重补齐。
- 成功补图时会额外注入一条不写入历史的临时说明，让主模型知道当前引用图片已经作为本轮视觉输入提供。
- 该能力只处理当前消息引用图，不恢复历史图片、不展开群友合并转发内部图片；与优化图片历史上下文、更好的图像转述和群聊上下文优化保持独立。

## 1.2.6

- 修复群聊上下文优化中压缩模型读取截断前完整主会话历史的问题：压缩前会按 AstrBot 当前 `max_context_length` / `dequeue_context_length` 规则预裁剪一份历史副本。
- 该预裁剪只影响传给压缩模型的 prompt，不修改 `req.contexts` 本体，主模型正式截断、历史保存和工具链仍由 AstrBot 原生流程处理。
- 配置提示补充说明：群聊滚动窗口数量继续沿用 `group_message_max_cnt`，主会话最近历史不会全量交给小模型。

## 1.2.5

- 将群聊上下文压缩模型的等待超时时间从 45 秒调整为 300 秒，降低大上下文筛选时被过早中断的概率。
- 在群聊上下文压缩模型配置提示中补充缓存说明：压缩模型每轮输入都会随当前消息、主会话历史和群聊滚动窗口变化，提示词缓存通常很难命中，建议选择成本低、速度快的小模型。

## 1.2.4

- 新增优化图片历史上下文：每轮 LLM 请求前清理会话历史里的旧图片 base64，并在保存历史前阻止新的 `data:image/...;base64` 图片块落库。
- 历史图片会替换成轻量占位符，普通文本、图片说明和其他上下文保持不变，避免最近 30 轮里旧图片反复撑爆主模型和小模型 token。
- 当前消息新发图片和当前引用图片仍按 AstrBot 原逻辑进入视觉模型或图片转述流程。

## 1.2.3

- 增强群聊上下文优化：开启后复用 AstrBot 自己维护的最近 N 条群聊缓存作为滚动窗口，N 仍完全沿用 AstrBot `group_message_max_cnt` 设置。
- 调整 AstrBot 群聊上下文注入链路：压缩时不再消费删除已缓存群聊记录，连续触发 Bot 时仍能让小模型看到最近群聊窗口。
- 压缩失败、未选择模型或输出不合格时，不再把 AstrBot 原始群聊流水账回退注入给主模型，避免 token 爆炸。
- 压缩模型输入明确包含主会话最近历史、群聊滚动窗口和当前待回复消息；主模型只额外看到筛选后的相关群聊内容。

## 1.2.2

- 新增群聊上下文优化：当 AstrBot 自带群聊上下文感知启用时，可先用已配置的小模型筛选相关群聊原文并生成简短摘要，再临时注入给主模型。
- 压缩模型只负责筛选上下文，不生成回复建议；失败、未配置模型或当前没有群聊上下文时会自动回退 AstrBot 原始群聊流水账。
- 不新增群聊记录条数或会话历史轮数配置，全部沿用 AstrBot 当前设置。

## 1.2.1

- 新增实验性功能：自动报错分析与 Issue 助手。插件处理消息报错后，AstrNa 可读取脱敏 traceback，调用当前 LLM 做初步归因，并把提醒发送到配置的 Issue 助手通知/处理 UMO。
- 报错后续流程改为在绑定 UMO 中处理，AstrBot 管理员可兜底操作：可忽略、调用源码辅助分析、生成草稿、补充说明、取消或提交；同一能力也会注册为自然语言可调用工具。
- 支持生成 GitHub Issue 草稿；提交前会尝试读取目标仓库的 Issue 模板或 Issue Forms，并把脱敏日志、AI 分析、源码分析和用户补充说明填入草稿。
- 新增可选子开关“提供阅读源码和修改源码的功能”，用于联动弥亚开发工具箱至少 `2.6.0` 版本；源码辅助提示只在用户确认分析后临时注入，不写入会话历史。
- 用户确认前不会提交 Issue；GitHub Token 只用于 GitHub REST API 提交，不会发送给 LLM、工具提示或 Issue 正文，AstrNa 自身日志会尽量脱敏。
- 第一版优先处理 AstrBot `on_plugin_error` 能捕获的插件报错；Core 级错误会提示用户临时开启 DEBUG/文件日志后补充复现信息。

## 1.1.9

- 新增实验性开关：解锁群聊并发回复。开启后，同一群内不同群友可以并发触发 Bot 回复，同一群友和私聊仍保持串行。
- 兼容消息防抖插件的按发送者防抖 key，优先保留连续消息合并和取消旧回复语义。
- 增加并发历史保存保护，降低同群多用户并发回复时互相覆盖会话历史的风险。

- 重写 README，补充正式版功能说明、安装方式、兼容性、验证状态和各开关使用边界。
- 当前正式版包含 DeepSeek V4 400 兼容修复、身份元数据优化、群身份/生日工具、图像转述优化、send_message_to_user 优化、合并转发优化、超长回复上下文优化、动态 system prompt 缓存优化和回复指向说明等能力。

## 0.1.8

- 新增优化超长回复上下文模块：Bot 的 LLM 长回复被 AstrBot 或输出插件转换为合并转发、或分段后只剩尾段发送时，会尽量把完整纯文本补回后续会话上下文。
- 该能力只处理 Bot 自己的 LLM 回复，不展开群友发送的合并转发消息，避免把别人的合并转发内容误塞进上下文。

## 0.1.7

- 增强提供群身份查询工具模块：新增查询当前群某个成员生日月日的工具，并可查询当前群未来一段时间内过生日的成员列表。
- 生日查询继续限制在当前群内，不支持跨群查询；只返回月日，不返回年份，查不到时会结构化返回失败原因。

## 0.1.6

- 增强优化身份元数据模块：新增可选的注入生日信息子开关，会在 AstrBot 自带身份识别已生效时，通过 NapCat/aiocqhttp 尝试补充当前聊天对象的 QQ 生日月日。
- 生日信息只注入月日，不注入年份；查不到、字段为空或被平台隐藏时会自动跳过，并继续作为临时身份元数据处理。

## 0.1.5

- 增强 DeepSeek v4 400 修复模块：同一个开关现在会同时清理异常 assistant 历史，并修复代理模型名下 DeepSeek V4 thinking mode 未回传 `reasoning_content` 导致的 400。
- 支持 `deepseek-v4`、`deepseek-v4-pro`、`deepseek-v4-flash` 及带代理前缀的同名模型，避免误处理 `deepseek-chat`、`deepseek-reasoner`。

## 0.1.4

- 优化回复历史标记改为临时中文自然语言提示，不再长期依赖机器标签来区分回复对象。
- 功能关闭时也会清理旧的 AstrNa 内部标记，避免历史污染继续影响模型。

## 0.1.3

- 修复引用 Bot 历史回复后，模型可能把“被引用回复原本回复的人”误当成当前发言人的问题。
- 优化回复历史标记模块会在发给模型前临时清理旧历史里的 AstrNa 内部标记，并移除模型误输出的内部标记，避免污染真实发送内容和后续历史。
- 保存新历史时会先移除旧内部标记，再写入当前事件生成的权威回复目标标记；引用 Bot 回复时优先使用可信索引或唯一历史匹配。

## 0.1.2

- 修复优化回复历史标记模块中，回复目标标记可能排在 assistant 思考内容之后，导致模型优先读取旧思考而认错当前用户的问题。
- 引用 Bot 历史回复时，如果能从当前会话历史中安全匹配到原回复记录，会额外补充这条 Bot 回复原本回复给谁。

## 0.1.1

- 新增优化回复历史标记模块：保存 LLM 回复历史时为 assistant 回复补充回复目标标记，帮助群聊上下文区分回复对象。
- 引用消息会补充引用发送者标记；标记只进入新历史记录，不改变真实发送到聊天平台的内容。

## 0.0.8

- 新增提供群身份查询工具模块：为 Bot 提供按需查询当前群成员身份、群等级、专属头衔、群主和管理员的工具。

## 0.0.6

- 跨会话主动消息、定时任务、图片、文件、语音、视频和 @ 等场景继续走 AstrBot 原逻辑。

## 0.0.5

- 保持 AstrBot 原有图片压缩、转述结果写回和多模态 fallback provider 逻辑不变。

## 0.0.4

- 新增 AstrBot插件缓存优化模块：观察其他插件对 `system_prompt` 的动态追加，在同一注入位置连续 3 次确认安全后持久化接管，并将后续完整动态语义块迁移到临时 `extra_user_content_parts`，减少动态提示词对 LLM 缓存命中的影响。
- 固定不变的提示词会继续保留在 `system_prompt`；动态内容会尽量作为完整语义块迁移；复杂重写、删除或替换会自动跳过。

## 0.0.3

- 新增优化合并转发模块：在 AstrBot 自带合并转发已经触发时，将过长的单个转发节点拆成多个较短节点，降低 QQ/NapCat 因单节点过长发送失败的概率。
- 新增单个转发节点目标长度和硬上限配置项，默认分别为 `1000` 和 `1200`。

## 0.0.2

- 新增真实昵称模式：支持追加真实昵称，或在可用时仅使用真实昵称。
